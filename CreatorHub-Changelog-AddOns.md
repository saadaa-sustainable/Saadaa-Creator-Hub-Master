### Collab email send gate + Error Portal "Send again" (2026-07-09)
- **Hard gate — no incomplete email ever reaches a creator.** `sendCollabEmail` now resolves **both** required attachments (Campaign Brief + T&C) **and** confirms the sender CC *before* sending. If any is missing it **blocks the send entirely**, does **not** stamp `collab_email_sent_at`, and returns the reason to the onboarding modal (which already shows it inline + toast). Previously the send fired in a background `after()` and would ship without the brief/T&C if either failed to resolve.
- **Error Portal integration.** Blocked sends log `system_errors` type `collab_email_blocked` (SMTP failures → `collab_email_send_failed`), keyed by `post_id`. New **Email Blocked** KPI card (clickable → scrolls to the list) + a dedicated **Collab Emails Blocked** card showing each blocked collab (creator, campaign, status, the exact reason) with a **Send again** button. Retry (`resendBlockedCollabEmail`) rebuilds the email from the live preview — so a since-fixed brief / T&C / email is picked up — and re-runs the gated send; on success `resolveSystemError` clears the row and it drops off the portal.
- Files: `features/onboarding/actions.ts` (gate), `lib/system-errors.ts` (`resolveSystemError`), `features/errors/{actions,queries,types,page-client}.tsx`, Know More `errors`.
